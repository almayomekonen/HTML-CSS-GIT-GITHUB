# אוטומציה של בנייה והעלאה

המטרה במשפט אחד:

במקום לבנות ולהעלות את האימג' ביד בכל פעם, נגדיר שזה יקרה לבד אחרי כל דחיפת קוד.

כל פקודה ושם באנגלית מופיעים בבלוק נפרד.
ההסבר בעברית תמיד מתחת, בשורה נפרדת.

---

## מה זה בכלל

ראשי התיבות:

```text
CI/CD
```

הפירוש:

אינטגרציה רציפה ואספקה רציפה.
זה תהליך אוטומטי שלוקח את הקוד שלכם, בונה ממנו אימג', ומעלה אותו, בלי שאף אחד יקליד פקודות ביד.

הכלי שנשתמש בו:

```text
GitHub Actions
```

זהו שירות חינמי שמובנה בתוך גיטהאב.
הוא מריץ פקודות אוטומטית בכל פעם שדוחפים קוד למאגר.

---

## איך זה עובד בתמונה

```text
push code  ->  GitHub Actions  ->  build image  ->  push to Docker Hub
```

דוחפים קוד למאגר.
גיטהאב מזהה את הדחיפה ומפעיל את התהליך.
התהליך בונה אימג' חדש.
התהליך מעלה אותו לדוקר האב.
הכל לבד.

---

## הקובץ שמגדיר את התהליך

המיקום שלו במאגר:

```text
.github/workflows/docker.yml
```

חשוב לדייק בנתיב.
התיקייה מתחילה בנקודה.
גיטהאב מחפש את הקבצים האלה בדיוק במקום הזה.

---

## שלב ראשון — ליצור מפתח גישה בדוקר האב

לא משתמשים בסיסמה הרגילה באוטומציה.
יוצרים מפתח גישה ייעודי, שאפשר לבטל בכל רגע.

נכנסים לכתובת:

```text
https://app.docker.com/settings/personal-access-tokens
```

לוחצים על יצירת מפתח חדש.
נותנים לו שם, ובוחרים הרשאת קריאה וכתיבה.
מעתיקים את המפתח שמתקבל ושומרים אותו לרגע. הוא מוצג רק פעם אחת.

---

## שלב שני — לשמור את הסודות בגיטהאב

נכנסים למאגר בגיטהאב.
הולכים להגדרות, ואז לאזור הסודות.

הנתיב בתפריט:

```text
Settings -> Secrets and variables -> Actions
```

יוצרים שני סודות חדשים.

הסוד הראשון, שם המשתמש:

```text
DOCKERHUB_USERNAME
```

הערך שלו הוא שם המשתמש שלכם בדוקר האב.

הסוד השני, מפתח הגישה:

```text
DOCKERHUB_TOKEN
```

הערך שלו הוא המפתח שיצרתם בשלב הקודם.

---

## שלב שלישי — לדחוף את הקוד

מספיק לדחוף את הקוד כרגיל:

```bash
git add .
```

```bash
git commit -m "add ci cd workflow"
```

```bash
git push
```

מרגע הדחיפה, גיטהאב מתחיל לבנות ולהעלות לבד.

---

## שלב רביעי — לראות את זה רץ

נכנסים למאגר בגיטהאב.
לוחצים על הלשונית של התהליכים האוטומטיים.

שם הלשונית:

```text
Actions
```

רואים את התהליך רץ בזמן אמת, שלב אחרי שלב.
בסוף מופיע סימן ירוק.

---

## מה הרווחנו

לפני:

בנייה והעלאה ידנית בכל שינוי. שלוש פקודות בכל פעם, וקל לשכוח.

אחרי:

דוחפים קוד פעם אחת, וכל השאר קורה לבד. זה הלב של גישת הפיתוח המודרנית.
