# הסבר מלא — איך ארזנו את האפליקציה לדוקר

כל פקודה מופיעה בבלוק קוד בנפרד.
ההסבר בעברית תמיד מתחת, בשורה נפרדת.

---

## הפקודה שגרמה לקסם

```bash
docker run -d --name users-management-app -p 8080:3000 --env-file .env users-management
```

זו הפקודה המדויקת שהפעילה את הקונטיינר, ובזכותה אתה רואה אותו עכשיו בדוקר דסקטופ.

עכשיו נפרק אותה לחלקים. כל חלק בבלוק נפרד, וההסבר מתחתיו.

```bash
docker run
```

צור והפעל קונטיינר חדש מתוך אימג' קיים.

```bash
-d
```

הרצה ברקע. הקונטיינר רץ, והטרמינל משוחרר לשימוש.

```bash
--name users-management-app
```

נותן לקונטיינר שם קריא. בלי זה דוקר בוחר שם אקראי.

```bash
-p 8080:3000
```

מיפוי פורט. הפורט שמשמאל נמצא במחשב שלך. הפורט שמימין נמצא בתוך הקונטיינר.

```bash
--env-file .env
```

מזריק את המשתנים הסודיים לקונטיינר בזמן הריצה.

```bash
users-management
```

שם האימג' שממנו נוצר הקונטיינר.

---

## מה כל קובץ עושה

```text
Dockerfile
```

דף ההוראות לבניית האימג'. דוקר קורא אותו שורה אחרי שורה ובונה לפיו את התבנית.

```text
.dockerignore
```

רשימת קבצים שלא יועתקו לתוך האימג'. מונע העתקה של החבילות ושל קובץ הסודות.

```text
.env
```

מכיל את המשתנים הסודיים, כמו שם משתמש וסיסמה למסד הנתונים.

```text
app.js
```

הקובץ הראשי של האפליקציה. מפעיל את השרת ומתחבר למסד הנתונים.

```text
config/db.js
```

אחראי על החיבור למסד הנתונים בענן.

```text
README-DOCKER.md
```

מדריך צעד אחרי צעד לתלמידים.

---

## מה כל שורה בתוך ה-Dockerfile אומרת

זהו דף ההוראות עצמו. דוקר קורא אותו מלמעלה למטה ובונה את האימג' שורה אחרי שורה.

```docker
FROM node:20-alpine
```

מאיזה בסיס מתחילים. לוקחים אימג' מוכן שכבר מכיל את הסביבה של נוד בגרסה עשרים.
המילה שבסוף היא גרסה זעירה של לינוקס, ולכן האימג' יוצא קטן ומהיר.

```docker
WORKDIR /app
```

קובע את תיקיית העבודה בתוך הקונטיינר. כל הפקודות הבאות ירוצו מתוך התיקייה הזו.
אם היא לא קיימת, היא נוצרת לבד.

```docker
COPY package*.json ./
```

מעתיק קודם רק את קובצי רשימת החבילות לתוך הקונטיינר.
מעתיקים אותם ראשונים בשביל מנגנון הזיכרון. כל עוד הרשימה לא משתנה, דוקר מדלג על שלב ההתקנה בבנייה הבאה, וזה מאיץ אותה מאוד.

```docker
RUN npm install
```

מתקין את כל החבילות שהאפליקציה צריכה בתוך הקונטיינר.

```docker
COPY . .
```

מעתיק את כל שאר קוד האפליקציה לתוך הקונטיינר.

```docker
EXPOSE 3000
```

מצהיר שהאפליקציה מאזינה על פורט שלושת אלפים. זו הצהרה לתיעוד בלבד.
המיפוי האמיתי נעשה בזמן ההרצה.

```docker
CMD ["node", "app.js"]
```

הפקודה שתרוץ כשהקונטיינר עולה. כאן מפעילים את האפליקציה.
בכוונה לא משתמשים בכלי הפיתוח שמאזין לשינויים, כי בקונטיינר אין בו צורך.

## מה כל פקודה עושה

```bash
docker build -t users-management .
```

קורא את דף ההוראות ובונה ממנו אימג' בשם שבחרנו.
הנקודה בסוף אומרת שדף ההוראות נמצא בתיקייה הנוכחית.

```bash
docker images
```

מציג את כל האימג'ים הקיימים במחשב.

```bash
docker ps
```

מציג את הקונטיינרים שרצים כרגע.

```bash
docker ps -a
```

מציג את כל הקונטיינרים, כולל אלה שעצרו.

```bash
docker logs users-management-app
```

מציג את הפלט של האפליקציה מתוך הקונטיינר.

```bash
docker stop users-management-app
```

עוצר את הקונטיינר. הוא נעלם מרשימת הרצים, אבל עדיין קיים.

```bash
docker rm users-management-app
```

מוחק את הקונטיינר לגמרי.

---

## הרעיון המרכזי

```text
Dockerfile  ->  docker build  ->  IMAGE  ->  docker run  ->  CONTAINER
```

דף ההוראות הופך לאימג' בעזרת פקודת הבנייה.
האימג' הופך לקונטיינר חי בעזרת פקודת ההרצה.

המשל מהמטבח:
דף ההוראות הוא המתכון.
האימג' הוא התבנית הקפואה.
הקונטיינר הוא העוגה החיה שיצאה מהתבנית.
